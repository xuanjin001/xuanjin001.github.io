Getting your org ready for users

Org. Level:

So the company profiles is the top security control tool, and UI Setting is the second…

Company profile: A collection of information about your organization, mostly captured at contractual signup.

Company info:

` `

  • Name and address

` `

  • Primary contact

` `

  • Default locale

` `

  • Default company

` `

  • Storage used

` `

  • Licenses available

Financial Information: FY, Currencies

Support Information: Business hours, holidays

Locale Setting:

` `

  • Locale: (also in address format) determine how date, times, number and names are displayed

` `

  • Language: determine the language in the user interface and in Help and Training

` `

  • Time Zone: time zone used for event start and end times, and in Date/Time Fields

Each user has their own personal user record settings

Multi-Currencies:

Multi-currency allows just one exchange rate per currency by default!

Inline editing is not available for currency.

Create list views for frequently performed searches

` `

  • Save list views fro future use

` `

  • Filter on specific fields

` `

  • Specify which groups of users have access to the list view

` `

  • Print list views

` `

  • Edit records in list views

` `

  • Follow records and view related chatter posts

Users need creates and customize list view permissions in order to finish this (by default, they should have this function) can be removed

Chatter: Secure and works in accordance with all of the security and permission setting in SFDC org.

Chatter enabled: Account, Case, Chatter Group, Contact, Lead, Opportunity, Topic, User

Chatter Groups:

` `

  • Users can create groups

` `

  • Groups can be public private or unlisted

` `

  • Groups can be made “Broadcast Only”

` `

  • A group has only 1 owner and several managers

` `

  • The groups tab lists all groups

Notes:

` `

  • User interface & search settings are global settings that affect all users

` `

  • Default locale, language, and currency are set on the company profile, but can be modified for individual users

Setting up and managing users

Profile: collection of settings and permissions that determines what users can see in the user interface and what they can do

Settings(What users can see):

` `

  • Apps

` `

  • Tabs

` `

  • Record Types

` `

  • Page Layouts

` `

  • Fields

Permissions(What users do):

` `

  • Administrative

` `

  • General user

` `

  • Standard object

` `

  • Custom object

Profiles and Object Permissions (Profile = Permissions):

` `

  • Create

` `

  • Read

` `

  • Edit

` `

  • Delete

Tab Settings: define the tabs accessible to users after logging in

Tab Hidden: Allows you to hide a tab, the user cannot display it back

Default on: Allows you to display a tab by default.

Default off: Allows you to hide a tab by default. However, any users assigned to the profile can display it, if required.

Standard Profiles

` `

  • All Salesforce Editions have standard profiles

` `

  • Standard profiles cannot be edited and deleted

` `

  • Clone

System Admin: View and Modify all data, customize app

Standard User: View and edit records they can access and delete their own

Solution Manager: Standard user + manage published solutions

Marketing user: Standard user + import leads

Contract Manager: Standard user + manage contracts

Read only: only view records they can access.

Standard Chatter Profiles

` `

  • Are available based on Chatter Free or Chatter External user license

` `

  • Allow users to login to Chatter Only

Chatter Free User: Access all standard Chatter people, profiles, groups and files

Chatter Moderator User:

` `

  • Access same data as Chatter Free users

` `

  • Active/Deactivate other Chatter Free users/moderators

` `

  • Grant/revoke moderator privileges

` `

  • Delete posts and comments they can see

Chatter External User: Access groups they’ve been invited to and interact with members of those groups

User: Someone with login access to your salesforce org, who has license and a user record containing personal, security, and locale info.

` `

  • Personal: name, alias, email, phone/address, title

` `

  • Security and Access: username, license, profile, role, login history

` `

  • Locale: time zone, locale, language, currency

Chatter Free License

` `

  • Users can access people, profiles, groups, and files,

    ` `

    • Make posts

    ` `

    • View comments

    ` `

    • Upload and view files

    ` `

    • Join groups

Chatter External License

` `

  • Let people outside the company, such as customers, into private groups in the Chatter community

` `

  • Chatter external user can:

    ` `

    • See the groups they belong to, and the profiles of members of those groups

    ` `

    • Share files common to groups they belong to

Note: Chatter free users do not have access to any salesforce data for security reason

Chatter Customer Experience:

` `

  • Chatter Customers can only be invited to a private chatter group

` `

  • They can see:

    ` `

    • Groups and profiles of the members of groups they belong to

    ` `

    • Files shared to groups they belong to

` `

  • They cannot see:

    ` `

    • Post to profiles

    ` `

    • See records or other Salesforce information, even in search results

    ` `

    • Be followed. Additionally, customers can’t follow people or files

` `

Steps to trouble shoot login issues:

` `

  • Viewing a user’s log in history: Setup|Manage Users|Users|Name of the Userà show you the status of the password

` `

  • Dealing with an Invalid Password: users, check user, reset password

` `

  • Setup|Security Controls|PasswordPolicies confirming security setting

SalesforceA Can do:

` `

  • Freeze a user

` `

  • Deactivate a user

` `

  • Unlock a user

` `

  • Reset a user’s password

` `

  • View and edit a user’s information

` `

  • Assign permission sets

` `

  • View system status

` `

  • Switch between accounts in multiple orgs
