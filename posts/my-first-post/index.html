My First Post

As my first post, I want to talk about how I created this blog. There were definitely ups and downs. I cannot deny that, also there were frustrations.

Overall, I love this framework. I have searched online and see what frameworks are out there. The ones I looked at are Hugo and Hex, Hex seems to have more documentation on how each framework works and how you can utilize them. There is a huge community support as well. It is really great, however, most of the community support are in foreign language, and the amount that you can customize is a little limited. I also took a look at Hugo. Now, Hugo is different. It is powered by Go in the back-end. Go is an open source language, and it is really fast. This also result in that this that this framework works really fast. You can find out more about Go here

For this blog, I did the following.

  1. First of the FIRST: I went to read the documentation, that should be everyone’s first step, a good program should also come with a handy documentation. In the documentation, they explained how you can set up the environment, and how you can use this framework. Documentation

    When I was reading other blogs, I saw couple comments saying that Hugo doesn’t have enough documentation or explainations. When I first started out, the documentation is simple and straight-forward. HOWEVER, I have to say a BIG HOWEVER, there are a lot of things that’s not included in the documentation, you really have to hunt for it. For example, I followed the steps to set up the environment, that part was easy, but when I picked a theme and trying to customize the theme to make it to be fit for me, and more having a feeling for me. I was having a hard time to figure out how to do the customization. I guess what I read was right, it is hard to figure something out, but that’s also part of programmer’s life, isn’t it? You have a problem, you find a solution to resolve it. This post really helped me out, after I was stuck for a couple of days, I step upon this which totally helped me figure out how to fix the problem I have! Here is the blog.

  2. Let me explain the issues. The theme I picked is ‘monopriv’. It has 4 buttons on top of the page, where it says “Home”, “About”, “Blog”, “Tags”. I couldn’t figure out how everything is linked together. My first thought was, this definitely has something to do with “router”, which I absolutely hate. When I was trying to figure out Vue, I couldn’t figure out how all of the pages were linked, this has always been my problem. I am relatively new to the whole front-end piece, to be honest, but I am catching up. If you have been in this industry long enough, you would know that there is a magic thing called “Inspect Elements”!!! I still remember the first time, I actually understand this, it is just amazing! You can use it to find the EXACT element on your page, and you can even try to modify on the spot to see the changes, then change that in your code. I mean testing for front-end does takes a lot of time and it could get really annoying, but trust me, in the end it will be all worth it! So that’s how I found out that the original theme was having a direct link for those buttons. However, I couldn’t understand where are they hosting those pages. This is where the blog I mentioned earlier comes in, it mentioned that, they are hosting everything under the “content” folder. AND that worked!!! Everything just fall through later.

  3. Another wall I hit was to run this on a github hosting. THANKS to google, you will need to first deploy in your local environment, then push the changes into your github page.

Steps:

  1. Install Hugo
  2. Create a New Site
  3. Add a Theme
  4. Add some content (only the blog posts)
  5. Start Hugo Server
  6. Customize the Theme
  7. Build basic static pages All of the above steps you can find more details here
  8. Cd into the public folder
  9. push those changes into github pages

In the end, this is really easy to set up, and your blog will come out beautiful.

You should give it a try!!
